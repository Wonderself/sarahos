# ADR-001: Multi-Agent Tripartite Architecture (L1/L2/L3)

## Status
ACCEPTED

## Date
2026-02-27

## Context
SARAH OS requires an architecture that can scale from simple task execution to autonomous enterprise management. A single monolithic agent would be too prone to hallucination and impossible to debug. A flat swarm of agents would lack coordination.

## Decision
Adopt a 3-level hierarchical agent architecture:
- **Level 1 (Execution)**: Specialized agents with narrow, granular prompts for specific tasks. Use fast/cheap models (claude-sonnet).
- **Level 2 (Management)**: Manager agents that decompose L3 directives into atomic L1 subtasks, verify dependencies, and compile results.
- **Level 3 (Executive)**: Strategic agents with holistic system visibility, using advanced models (claude-opus) for complex reasoning.

## Rationale
- Small specialized prompts are significantly more accurate and less prone to hallucination than one massive orchestrating prompt
- Hierarchical structure mirrors real enterprise organization
- Each level can use appropriately-sized (and priced) LLM models
- Isolation enables independent testing, debugging, and replacement of agents
- Clear escalation paths from L1 → L2 → L3 → Human

## Consequences
- More complex infrastructure to build and maintain
- Inter-agent communication overhead
- Need for robust event bus and agent registry
- Higher initial development effort, but dramatically better scalability and reliability
