# ADR-002: TypeScript Strict Mode for Agent Type Safety

## Status
ACCEPTED

## Date
2026-02-27

## Context
SARAH OS agents communicate via structured JSON messages. Malformed payloads between agents could cause cascading failures across the entire system. Runtime type errors in a multi-agent system are extremely difficult to debug.

## Decision
Use TypeScript with `strict: true` and zero tolerance for the `any` type. All inter-agent messages, events, and API payloads must be fully typed. Use Zod for runtime validation at system boundaries.

## Rationale
- Compile-time type checking catches payload mismatches before runtime
- TypeScript strict mode enforces exhaustive null checks, preventing null reference errors
- Zod provides runtime validation for external inputs (API calls, user input, LLM responses)
- The combination of compile-time + runtime validation creates a robust safety net
- Node.js non-blocking I/O model is ideal for agents waiting on API responses

## Consequences
- Slightly slower initial development due to type definitions
- All developers must understand TypeScript strict mode
- Every new event type or message format requires type definitions
- No shortcuts — `any` is never acceptable
