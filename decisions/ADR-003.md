# ADR-003: Presets + Name for Client Avatars

## Status
ACCEPTED

## Date
2026-02-27

## Context
Clients need personalized AI assistants. Options considered:
1. Full custom avatar per client (expensive — unique D-ID sessions, voice cloning, etc.)
2. Two base avatars (Sarah/Emmanuel) with client-customizable name and persona prompt

## Decision
Use two base avatars (Sarah = female, Emmanuel = male) as shared presets. Clients choose a base, assign a custom name, and configure persona parameters (company name, industry, greeting, tone). The face and voice remain the same across all clients using the same base.

A "Custom Avatar" tier is planned for the future (waitlist placeholder now, no implementation).

## Rationale
- 100 clients with 2 bases = 2 D-ID caches → ~$. 100 clients with custom avatars = 100 caches → ~$$$$$
- Approximately 50x cost reduction via cache mutualization
- Only the name and prompt change between clients (nearly free in tokens)
- IP is 100% owned (both avatars derived from founder's image)
- Zero licensing risk, zero revocability risk
- Custom avatar tier can be added later as premium upsell

## Consequences
- All clients sharing a base have the same face and voice
- Client differentiation is limited to name, greeting, tone, and company branding
- "Coming Very Soon" section needed on site/app with waitlist for custom avatars
- Database must include `avatar_type` field ready for future "custom" type
